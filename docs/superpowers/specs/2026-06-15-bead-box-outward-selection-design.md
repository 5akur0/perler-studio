# 豆盒外扩选中框设计

## 目标

豆盒色块的选中框统一向色块外侧扩展，不再侵占色块内部空间。桌面端、移动端和绘图台使用同一套选中视觉。

## 选中样式

- `.color-chip.active` 使用 `2px` 深色 `outline`。
- `outline-offset` 为 `2px`，让轮廓与色块边缘之间保持清晰间隔。
- 保留一层低透明度外部晕圈，减轻纯黑硬边感，同时不削弱选中识别。
- 选中态不改变色块尺寸、内部 padding 或色样面积。

## 状态组合

- 普通选中色块使用统一外描边。
- `.active.needed` 保留“本图所需颜色”的品牌色辉光，但选中轮廓仍以深色外描边为最高优先级。
- `.held` 继续表达镊子持有状态；本次不改变其语义或颜色。
- 键盘 `:focus-visible` 仍使用品牌色焦点环，与深色选中框保持可区分。

## 容器与裁切

- 继续使用 `.color-palette` 的滚动容器内边距为外描边预留空间。
- 移动端维持至少 `4px` 内边距。
- 桌面端沿用 `--scroll-pad`，确保第一列、最后一列和顶部色块的外描边不被 `overflow: auto` 裁切。
- 底部渐隐遮罩保持不变；验证滚动到底部时选中框不会出现异常硬切。

## 实现范围

- 修改源样式 `src/styles/components.css` 和必要的响应式覆盖。
- 同步更新 `DESIGN.md` 与 `design-system/MASTER.md` 中关于色号 chip 选中态的说明。
- 重新构建生成的 `styles.css`。
- 不修改豆盒交互逻辑、颜色数据、布局列数或动画节奏。

## 验证

- 桌面豆盒：普通、边缘、所需色选中态。
- 移动豆盒：普通、边缘、所需色选中态。
- 绘图台最近颜色与完整色板。
- 检查外描边不挤压色样、不被容器左右和顶部裁切，并与焦点态可区分。
