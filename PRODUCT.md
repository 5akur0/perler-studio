# Product

## Register

product

## Users

主要用户是**治愈系休闲玩家**：喜欢慢节奏手作 / 模拟经营类小游戏、想放松解压的人，多在手机上利用碎片时间游玩（移动优先、强制竖屏）。围绕这一核心，还有三类重要用户：

- **小红书 / 社交平台内容创作者**——为了产出好看的拼豆成品图与分享文案，期望「每一屏都能单独截图」。
- **真实拼豆爱好者**——现实里玩 perler / hama beads，想先在数字端预览配色、试图纸（221 色 MARD 色卡、可改尺寸）。
- **桌面端深度玩家**——在电脑上认真做大尺寸图纸（最大 100×100）、导入自定义图片精修配色（强制横屏）。

使用场景横跨「手机上随手放松一会儿」到「电脑前慢慢做一幅大作品」，UI 必须同时服务这两端的节奏与输入方式，但**不要求两端复刻同一套工具布局**。

## Product Purpose

在纯前端、可 `file://` 双击离线运行的网页里，模拟从选图到收藏的完整拼豆体验。平台分工如下：

- **手机端：轻量摆豆流程**——选图 → 直接选取本图颜色 → 摆放 → 检查 → 熨烫 → 冷却 → 收藏。手机不承载豆筛、豆针、镊子等占空间的桌面工具；手作感由取豆、落豆、熨烫等反馈承担。
- **桌面端：完整手作流程**——选图 → 取豆 → 豆筛整理 → 使用豆针 / 镊子摆放 → 检查 → 熨烫 → 冷却 → 收藏。

这里的「选图」由**图纸库**承担：它是拼豆台新开作品时的挑图屏（不是独立入口），装着默认 + 用户导入的图纸，可星标置顶、删除、恢复默认、拼音排序，点一张直接开摆，分享码导入按钮固定在底部。**绘图台**是另一拨用户的「创作 + 分享」工具（自由绘制 / 图片转图纸 / 生成分享码），画完可存进图纸库；**图纸画廊**仍是独立的云端社区浏览，三者职责不同。

两端共享同一件作品、同一套阶段语义和情绪体验，但交互密度按屏幕与输入方式适配。它不是一个抽象的像素编辑器，而是一套有质感、有温度的「桌前慢慢做手工」体验。

成功 = 四件事同时成立：

1. **完成的治愈感**——玩家能把一幅作品从头做完，过程平静、收尾有成就感（情绪体验优先）。
2. **可分享 / 出圈**——成品与每一屏都「出片」，驱动用户自发截图分享，带来传播。
3. **手作真实感**——桌面端忠实还原倒豆、抖筛、工具摆放、熨烫与压平；手机端保留落豆、检查、熨烫与完成反馈，不以缩小桌面工具换取形式上的一致。
4. **上手无门槛**——零学习成本，新玩家几分钟内就能跑通完整流程，不靠说明书。

## Brand Personality

三词人格：**治愈 · 温润 · 沉浸**。

- **语气 / 文案**：中文为主，温柔、口语化、带一点手作的仪式感（参考 toast / 提示文案与近期 `polish:` 提交风格）。
- **情绪目标**：柔和粉彩、拟物温度、氛围感；让人想起「桌前慢慢做手工」的安心与专注，而非完成一个任务的紧迫。
- **质感取向**：圆润、轻盈、黏土般柔和的阴影与玻璃面，刻意为「氛围感」做高透取舍（见 `design-system/MASTER.md`）。

## Anti-references

> 视觉层面的硬约束已在 `design-system/MASTER.md` §0 落地，这里是策略层的「绝不像」：

- **商业 SaaS 冷淡风 / 仪表盘感**——高对比硬朗、冷灰蓝、信息密度堆叠、效率至上的工具气质。
- **「生成感」无氛围**——一眼 AI 的同质化卡片网格、薄荷绿写死不随主题、缺少手作温度。
- **emoji 当图标**——一律 SVG（`stroke="currentColor"`），不用 emoji 充当 UI 图标。
- **跨注册通病**：渐变文字、装饰性玻璃滥用、彩色侧边竖条、千篇一律的等大卡片网格、每个区块顶部的小号大写 eyebrow。

## Design Principles

1. **截图即门面（Every screen is a portrait）**——把每个阶段都当作一张可独立成立的静帧来设计；「出片 / 可分享」是一等结果，不是事后处理。
2. **拟物的温度（Warmth through skeuomorphism）**——桌面端用倒豆、抖筛、工具摆放、熨烫承载完整手作感；手机端用取豆、落豆、检查和熨烫反馈承载触感，不强塞豆筛与工具面板。
3. **治愈胜过效率（Calm over speed）**——保护慢而冥想的节奏；绝不为了让玩家更快做完而把「慢慢做」的乐趣优化掉。
4. **零门槛沉浸（Frictionless immersion）**——新手应能凭直觉跑通整条流水线；靠可供性与温柔反馈来教学，而非说明书或弹窗轰炸。
5. **一套语言贯穿到底（One language, every theme）**——治愈身份必须在全部 5 套主题、4 套工具款式、任意背景图上都成立；一致性来自 token / MASTER 这个单一事实来源，而非逐屏的一次性特例。

## Platform Interaction Strategy

- **手机拼豆色板**：只显示当前图纸实际使用的颜色，优先保证棋盘、颜色选择和主操作处于拇指可达区域。
- **电脑拼豆色板**：保留完整 MARD 221 色，本图用色以高亮状态标识；这是桌面深度玩法，不视为冗余。
- **绘图台色板**：保留完整 MARD 221 色，服务真实拼豆预览、自由绘制和精修配色。
- **工具差异**：豆筛、豆针、镊子仅在具备足够空间的桌面拼豆流程出现；手机端直接摆豆是正式产品策略，不是功能降级或待补缺口。
- **手机棋盘是工作视窗，不是全览缩略图**：初始状态不要求一次显示完整棋盘；用户可以自然地缩放、平移到任意有效格子。
- **可达性优先于强制全览**：成功标准是所有有效格子都能被看见并操作，而不是把整块棋盘强行缩小。不得为了初始全览牺牲豆子辨识度、落点精度或沉浸感。
- **跨端取景可以不同**：手机、平板和桌面可以使用不同的初始缩放、取景和工具密度，但必须共享作品状态与阶段语义。

## Accessibility & Inclusion

目标 **WCAG AA**（硬清单见 `design-system/MASTER.md` §1、§7）：

- **对比度**：正文 / 小字 ≥ 4.5:1（`--ink`、`--muted` 均达标）；大字（≥18.66px 粗体 / ≥24px）≥ 3:1。**有意例外**：主按钮白字 + cta 渐变 ≈ 3.3:1（满足大字 3:1，不满足正文 4.5:1），为「更亮观感」的刻意取舍，且每套主题各自压深保证 ≥3:1——cta 色不得用于小字。
- **颜色非唯一信息载体**——五彩强调色（分类用）必须配文字 / 图标。
- **动效**：全部动画（含环境型无限循环）须被 `prefers-reduced-motion: reduce` 覆盖；JS 侧用 `prefersReducedMotion()`（`utils.js`）。
- **焦点 / 语义**：所有交互元素有可见 `:focus-visible` 焦点环；图标按钮有 `aria-label`、图片有 `alt`；切屏时非活动屏 `aria-hidden`。
- **触控 / 输入**：触控目标 ≥ 44×44px；输入框 ≥ 16px 防 iOS 聚焦缩放。
